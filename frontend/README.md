# IRIP — Intelligent Risk Intelligence Platform (AI Security System)

A state-of-the-art React frontend built to act as the central nervous system for the **IRIP AI Security System**. This dashboard provides real-time visibility into network traffic, machine-learning evaluated risk scores, live firewall rules, and intrusion detection analytics.

## 🚀 Tech Stack

- **Framework:** React 19 + Vite + TypeScript (Strict Mode)
- **Styling:** Tailwind CSS (Dark Mode optimized for SOC environments)
- **Icons:** Lucide React
- **Data Visualization:** Recharts
- **Routing:** React Router v6
- **API Communication:** Axios (Token-based JWT Authentication)

---

## 🏛️ Architecture & Core Pages

### 1. `Dashboard.tsx` (Command Center)
The primary landing experience for administrators.
- **Top Metrics:** Displays aggregated real-time database statistics (total API events, active risk thresholds).
- **Live ML Risk Score Timeline:** Uses an `AreaChart` from Recharts to dynamically map the Risk Scores (0-100) generated by the Python Isolation Forest ML model.
- **Audit Feed Widget:** Continuously streams recent security audit events from the PostgreSQL database, color-coding high-risk behaviors and locks.

### 2. `Firewall.tsx` (Network Gatekeeper)
A comprehensive CRUD interface mapping directly to the Spring Boot backend `/api/firewall` routes.
- Renders active firewall rules blocking malicious IPs, User Agents, and Payload patterns.
- Analysts can manually **Toggle** a rule's active state or instantly **Drop** it entirely from the database.

### 3. `IDS.tsx` (Intrusion Detection Simulator)
A deep-dive tool allowing Security Analysts to perform manual pattern matching.
- **Threat Sandbox:** Allows administrators to test an IP Address and JSON request payload against the Python ML detection engine without executing malicious code on the server.
- Provides raw JSON dump views of the parsed output (e.g., detecting `' OR 1=1 --`).

### 4. `Users.tsx` (Identity & Access Authority)
The master dashboard for handling potential brute-force lockouts.
- Surfaces every registered user alongside their **Failed Login Count** and **Lock Status**.
- Connects directly to the Admin API to execute instant Account Unlocking for falsely flagged accounts.

## ⚙️ Running Locally

1. **Install dependencies:**
    ```bash
    npm install
    ```
2. **Start the Vite Dev Server:**
    ```bash
    npm run dev
    ```
3. **Important:** The dashboard expects the Spring Boot Backend API to be running on either `http://localhost:8080/api` or the defined environment base URL. See `/src/api/client.ts`.

---

## 🛡️ Security Implementation
- **JWT Protection:** The entire `Layout.tsx` relies on a valid Bearer token stored in LocalStorage. If the backend revokes token validity or drops the connection, the layout violently wipes local cache and forces re-authentication.
- **Clean Architecture:** No hardcoded secrets or sensitive environmental keys exist in the frontend source code.
